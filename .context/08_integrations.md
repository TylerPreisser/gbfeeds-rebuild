## B. Phase 9 Deploy Checklist (Tyler's ordered runbook)

The integrations above leave **15 items live** at build time and **9 items requiring deploy-time action** (most of which are env vars + Worker secrets — no code changes). Greg has 1 ongoing operational task (`harvests.json`).

The order below is dependency-strict. Skipping a step or doing it out of order will cause a soft failure (forms 4xx, missing badge, broken Stripe click, etc.) but no security incident. Steps 1–4 must precede 5; 5 must precede 6; 6–8 are independent of each other and can run in parallel; 9–10 are post-launch verification.

1. **Resend account + domain verification** — Sign up at resend.com. Add `gbfeeds.com` as a sending domain. Add the SPF + DKIM TXT records to GB Feeds' DNS (whichever DNS host is authoritative — GoDaddy until Step 7, then Cloudflare). Wait for green checkmarks. Generate an API key with send permission.

2. **Cloudflare Turnstile site provisioning** — In the Cloudflare dashboard → Turnstile → Add site. Domain pattern: `gbfeeds.com` + `*.gbfeeds-rebuild.pages.dev`. Save the **site key** (public) and **secret key** (private).

3. **Cloudflare KV namespace creation** — From the repo root: `cd cloudflare-worker && wrangler kv namespace create RATE_LIMIT_KV` then `wrangler kv namespace create RATE_LIMIT_KV --preview`. Paste the two returned IDs into `cloudflare-worker/wrangler.toml` (replace the two `REPLACE_WITH_*` placeholders). Commit the change.

4. **Worker secrets** — From `cloudflare-worker/`, run `wrangler secret put RESEND_API_KEY` (paste Resend API key from Step 1), then `wrangler secret put TURNSTILE_SECRET_KEY` (paste Turnstile secret from Step 2). These secrets are encrypted at rest in Cloudflare and are NEVER reachable from the Pages bundle, GitHub, or any local file.

5. **Worker deploy** — Confirm `TO_EMAIL = 'greg@gbfeeds.com'` in `cloudflare-worker/index.ts` matches Greg's verified inbox. From `cloudflare-worker/`, run `wrangler deploy`. Note the deployed URL (typically `https://gbfeeds-forms.<account>.workers.dev`). curl the Worker root path — should return CORS-aware 200 to a probe POST.

6. **Stripe Payment Links** — In the Stripe dashboard, create 16 SKU Payment Links + 48 bundle Payment Links (all one-time-payment, USD, fixed-price). Edit `src/data/products.live.json` (replace each `paymentLinkUrl` field, 16 places) and `src/data/payment-links.ts` (replace each entry in `productPaymentLinks` + `bundlePaymentLinks`, 64 places total). Commit + push. `npm run build` will fail in production mode if any `about:blank#TODO-` remains via `scripts/validate-products.ts`.

7. **Cloudflare Pages env vars** — In CF Dashboard → `gbfeeds-rebuild` → Settings → Environment variables → Production. Set:
   - `NEXT_PUBLIC_SITE_URL` = `https://gbfeeds.com`
   - `NEXT_PUBLIC_GA_ID` = `G-BF2FDR6KMM`
   - `NEXT_PUBLIC_TURNSTILE_SITE_KEY` = (Step 2)
   - `NEXT_PUBLIC_FORM_ENDPOINT` = (Step 5 Worker URL)
   - `NEXT_PUBLIC_TRUSTEDSITE_ID` = (Tyler's customer ID; leave unset if not yet recovered — badge will gracefully no-op)
   - `NEXT_PUBLIC_FEATURE_FIELD_CLUB` = `false`

8. **Pages deploy** — Push the Step 6 commit to `main`. CF Pages CI runs `npm ci && npm run build` and deploys `out/` to `gbfeeds-rebuild.pages.dev`. Verify the deploy is green; spot-check 5 routes returning 200.

9. **DNS cutover** — Repoint `gbfeeds.com` to Cloudflare Pages (apex A/CNAME records or full nameserver migration). Cloudflare handles edge serving + `_headers` + `_redirects` from this point forward. Wait for HTTPS cert provisioning (~minutes).

10. **Pre-launch sign-off** — From the live origin: `curl -I https://gbfeeds.com/` (HSTS + CSP present), submit `/sitemap.xml` to Google Search Console + Bing Webmaster Tools, end-to-end form submission test (`/contact` → email lands in Greg's inbox), Stripe Payment Link click-test on Buck Chow + one bundle, Google Rich Results Test on home + Buck Chow PDP + FAQ + a journal article. Phase 9 completes once all 10 boxes are checked.
